Mobile Content Management

Workspace ONE UEM REST APIs provide the ability to access and manage Personal Content. Features include adding, moving, sharing, and deleting Files and Folders.

Important Note: Personal Content APIs only support Basic Authentication. For more information, refer to the API Authentication section on the main Overview tab.
Personal Content APIs are divided into three parts:
  • Contents – Personal Content APIs perform operations to obtain information about the files and folder structure within Workspace ONE UEM Personal Content.
  • Files – Personal Content Files APIs perform operations to obtain information about the files and to perform actions on them.
  • Folders – Personal Content Folder APIs perform operations to obtain information about the folders and to perform actions on them.

Enabling Personal Content API Access

Workspace ONE UEM REST APIs enable your end-users to manage their Personal Content with features such as adding, moving, sharing, and deleting Files and Folders. Follow the below steps to make the most of this functionality:
  1. From the Workspace ONE UEM main menu, navigate to Groups & Settings > All Settings > System > Advanced > API > REST API.
  2. On the General tab, configure the following settings.

    3. Settings Description
    Enable API Access Select this check box to automatically generate the API key for the Organization Group
    Add Select Add to add/or generate multiple API keys.
    • Service – Enter one or multiple service(s) and generate their own independent API keys.
    • Account Type – Select the type of the account. To access Personal Content APIs (MCM APIs), select the Account type as Enrollment User.
    • Description – Enter short description for the service and generated API key.
    • Whitelisted Domains – Enter domains that can interact with Workspace ONE UEM APIs via API key.

  3. Navigate to Groups and Settings > All Settings > Content > Personal Content. Set Personal Content to Enabled to enable Personal Content at your organization group.
  4. Enable applicable Shared Links settings to use sharing file link functionality.
  5. Enable applicable Shared Folder settings to use sharing folders functionality.

For more information on Enabling and Configuring Personal Content, please refer to the Mobile Content Management Guide in Workspace ONE UEM Resources.
